# 104 推薦信產生教學

針對台灣 104 求職平台「自我介紹」欄位，產生可直接複製貼上的繁體中文推薦信。

---

## 前置條件

| 檔案 | 說明 | 建立方式 |
|------|------|----------|
| `cv.zh.md` | 繁體中文履歷（必要） | 手動建立，或告訴 Claude「幫我把 cv.md 翻譯成繁中版 cv.zh.md」 |
| `jds/{jobNo}.md` | 職缺 JD（建議事先抓取） | `npm run fetch104 -- {url}`（見下方） |

---

## 標準流程

### 步驟一：抓取 JD

```bash
npm run fetch104 -- https://www.104.com.tw/job/{jobNo}
```

- 不耗 AI token，直接呼叫 104 API
- 自動儲存至 `jds/{jobNo}.md`，包含公司簡介、職責、條件、福利
- 執行一次即可，下次直接跳過

### 步驟二：產生推薦信

在 Claude Code 互動 session 中說：

```
/career-ops coverletter {jobNo}
```

或自然語言：

```
幫我寫 {公司名} {職缺編號} 的 104 推薦信
```

Claude 會自動讀取 `cv.zh.md`、`jds/{jobNo}.md`、`config/profile.yml`，產生四段約 250–350 字的繁中推薦信。

### 步驟三：確認與修改

直接在對話中回饋：

```
第二段太客套，改成更直接的語氣
把某個專案例子拿掉
第三段動機寫得不夠具體，針對這家公司的產品方向多說一句
```

Claude 只修改你指定的段落，不整封重寫。

### 步驟四：複製貼上

確認後直接複製信件正文，貼入 104 應徵頁面的「自我介紹」欄位。

## 兩種執行模式

career-ops 的推薦信功能有兩套執行路徑：

| | 系統一：互動模式（建議） | 系統二：CLI 自動化 |
|---|---|---|
| **啟動方式** | `/career-ops coverletter {jobNo}` | `npm run coverletter -- {jobNo}` |
| **執行環境** | 目前 Claude Code session | 獨立 `claude -p` 子程序 |
| **互動** | 可即時回饋、修改段落、迭代 | 全自動，不能問問題 |
| **提示詞** | `modes/zh/coverletter.md` | `batch/coverletter-prompt.zh.md` |
| **輸出** | 在對話中顯示，你決定是否存檔 | 固定寫入 `output/coverletters/{jobNo}-{date}.md` |
| **適合場景** | 日常使用、需要調整品質 | 批次產出多封、不需要互動 |

**推薦用系統一**：推薦信要被真人讀到，第一版很少完美，即時回饋比自動化更重要。

## 推薦信結構

四段，繁體中文，總長 250–350 字：

| 段落 | 內容 | 長度 |
|------|------|------|
| **第一段（定位）** | 幾年資歷、核心專長、與此職缺最直接相關的身份 | 2–3 句 |
| **第二段（匹配）** | 對照 JD 主要職責，舉最相關的工作經歷或專案，用具體成果 | 3–4 句 |
| **第三段（動機）** | 為什麼對這家公司或職位有興趣，具體呼應 JD 的產品或技術方向 | 2–3 句 |
| **第四段（收尾）** | 期待進一步說明，不用空泛套語 | 1–2 句 |

**語氣守則：**
- 技術詞彙保留英文（React、Vue、CI/CD、LLM 等）
- 不用「熱愛學習」「積極主動」「抗壓性高」
- 不用第一人稱「我」開頭每一句
- 避免翻譯腔

## 資料來源規則

所有經歷、數字、技術**只能來自 `cv.zh.md`**，不可憑空填入。

| 來源 | 優先順序 |
|------|----------|
| `cv.zh.md` | 必讀（主要） |
| `jds/{jobNo}.md` | 必讀（若有） |
| `config/profile.yml` | 必讀 |
| `modes/_profile.md` | 必讀 |
| `reports/{num}-*.md` | 若存在，補充 fit 分析 |

## 常見問題

**Q：沒有 `jds/{jobNo}.md` 可以直接產生嗎？**
可以。有三種替代方式：
1. 貼上 JD 文字到對話中
2. 直接說公司名＋職稱（品質較差）
3. 提供 104 URL，Claude 會提示你先執行 `npm run fetch104`

**Q：`cv.zh.md` 怎麼建立？**
如果已有 `cv.md`（英文），在 Claude Code 中說：
```
幫我把 cv.md 翻譯成繁中版，存成 cv.zh.md，術語保留英文
```

**Q：可以一次產生多封推薦信嗎？**
可以用系統二批次執行：
```bash
npm run coverletter -- {jobNo1}
npm run coverletter -- {jobNo2}
```
但品質不如逐一在系統一互動調整。

## 輸出格式

```markdown
## {公司名稱} {職稱} — 推薦信

{信件正文，直接可複製}

---
字數：{N} 字
來源：cv.zh.md + jds/{jobNo}.md
日期：{YYYY-MM-DD}
```

系統二的輸出固定存於 `output/coverletters/{jobNo}-{date}.md`。
系統一的輸出在對話中顯示，你可以請 Claude 存成指定路徑。
