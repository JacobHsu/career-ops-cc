# 履歷診斷與優化：使用教學

`optimize` 模式讓 AI 扮演資深招募主管，對你的履歷做全面診斷並提供具體改寫建議。

---

## 前提：確認這三個檔案已設好

| 檔案 | 內容 | 如何確認 |
|------|------|----------|
| `cv.md` | 你的完整履歷 | 打開看是否有內容 |
| `config/profile.yml` | 姓名、目標職缺、薪資目標 | 確認非範本預設值 |
| `modes/_profile.md` | 個人定位與原型 | 確認有你的實際描述 |

如果任一缺少，先說「幫我設定 profile」讓系統引導你完成。

---

## 基本用法

直接說白話，不需要特殊指令：

```
幫我診斷履歷
優化我的 CV
我的履歷有什麼問題？
```

---

## 進階用法

### 指定目標職缺方向

```
幫我針對「前端工程師」診斷履歷
幫我優化履歷，目標是外商技術職
我在找 full-stack 工程師，先幫我診斷缺什麼
```

### 搭配 JD 一起分析

貼上職缺描述，讓診斷更精準：

```
幫我診斷履歷，這是我目標的 JD：
[貼上職缺描述]
```

或貼 URL：

```
幫我針對這個職缺診斷我的履歷：
https://jobs.example.com/frontend-engineer
```

### 只針對特定部分

```
只幫我重寫 Professional Summary
幫我優化工作經歷的 bullet points
我的 ATS keyword 夠嗎？
```

---

## 輸出內容

系統會產生包含以下六個區塊的診斷報告：

| 區塊 | 內容 |
|------|------|
| A. 整體評分 | 1–5 分、ATS 通過率預估、主要問題清單 |
| B. Summary 重寫 | 嵌入 keyword 的新版 summary |
| C. Bullet 優化 | 每條 bullet 的改寫建議 |
| D. ATS Keyword | 缺少的關鍵字與建議嵌入位置 |
| E. 履歷 Gap | 時間斷層、技能缺口、缺量化數字的段落 |
| F. 職缺方向建議 | 最適合你的 2–3 種職缺類型與理由 |

報告自動存到 `reports/cv-optimize-YYYY-MM-DD.md`。

---

## 診斷後的下一步

診斷完成後，可以接著說：

```
# 把優化建議寫回 cv.md
幫我把這些改寫更新到 cv.md

# 針對特定職缺產生 PDF
幫我用這份優化後的 CV 產生 PDF
```

---

## 使用時機

| 情境 | 建議 |
|------|------|
| 剛開始求職 | 先做一次完整診斷 |
| 投了很多但沒回音 | 做 ATS keyword 專項分析 |
| 要應徵特定職缺 | 貼 JD 一起診斷 |
| 換產業或轉職 | 重點看 Gap 與職缺方向建議 |
| 定期維護 | 每 1–2 個月做一次 |
