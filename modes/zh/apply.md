# 模式：應徵表單與自我推薦

用於 104、CakeResume、Yourator、Meet.jobs 或公司官網表單。

永遠不要替使用者按下送出。只產生可複製的內容，最後由使用者確認。

## 流程

1. 讀取目前表單、截圖或使用者貼上的問題
2. 辨識公司與職稱
3. 搜尋 `reports/` 是否已有評估
4. 讀取 report、`cv.md`、`config/profile.yml`、`modes/_profile.md`
5. 產生繁中或中英混合回答
6. 標出需要使用者確認的資訊

## 常見欄位

- 自我推薦
- 為什麼想加入
- 相關經驗
- 期望薪資
- 可到職日
- 遠端/混合/通勤
- 是否接受派遣、駐點、約聘
- 作品集連結
- 補充說明

## 自我推薦語氣

台灣平台建議：

- 直接但不浮誇
- 不使用過度行銷語
- 技術詞保留英文
- 先講匹配，再講證據
- 避免「熱愛學習」「抗壓性高」這類空泛字

結構：

1. 一句定位：Senior Frontend / Full-stack / React / Vue
2. 一個最相關經驗
3. 一個作品或專案證據
4. 一句為什麼適合該職缺

## 薪資回答

如果 `config/profile.yml` 的薪資是 `TBD`：

- 不要編造數字
- 建議使用區間式回答
- 請使用者確認目標薪資

範例：

> 期望薪資會依職務範圍、遠端/混合型態、整體年薪與福利結構討論。若方便的話，也希望先了解貴公司此職缺的預算區間。

## 輸出格式

```markdown
## {Company} -- {Role} 應徵回答

依據：Report #{num} | Score {X.X}/5

### 1. {表單問題}
> {可複製回答}

### 2. {表單問題}
> {可複製回答}

需要確認：
- {薪資 / 到職日 / 通勤 / 授權等不確定資訊}
```

## 投遞後

使用者確認已投遞後，才可把 tracker 狀態改為 `Applied`。
